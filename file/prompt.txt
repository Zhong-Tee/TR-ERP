ช่วยวางแผนเพื่อสร้างโปรเจ็ค React ใหม่ 
ชื่อโปรเจกต์ TR-ERP โดยเชื่อมต่อกับ Supabase ในการเก็บข้อมูลทั้งหมด
ส่วนการ Login ใช้ Authentication และสร้างตาราง users เพื่อจำกัดสิทธิ์การเข้าถึงเมนู
มีการเชื่อมต่อ EasySlip API สำหรับเมนูสร้างออเดอร์ เพื่อตรวจสลิปอัตโนมัติ 
(url document : https://document.easyslip.com/documents/verify/bank/image)
ต้องทำให้รองรับการเพิ่มรูปสลิปหลายรูป รวมยอดจากสลิปโอนหลายใบ
และใช้วิธีการ loop through each image and verify 
และหากยอดเกินให้มีการแยกรายการไปอีกเมนูย่อยสำหรับให้แผนกบัญชีทำเรื่องโอนคืนลูกค้า โดยมีระบบป้องกันไม่ให้ใช้สลิปซ้ำ
ต้องไม่เก็บ API Key ไว้ที่ fornt_end หรือ ทำให้ปลอดภัยเท่านั้น

ช่วยแนะนำขั้นตอนแบบ step by step ให้ด้วย
ความต้องการของโปรเจกต์นี้คือ
ใช้สำหรับลง order และ ตรวจสอบสลิปโอน
หากตรวจสลิปผ่านจะให้ไปแสดงรายการใบงานได้
และเข้าสู่กระบวนการ QC เมื่อ QC Pass ฝ่ายแพ็คสินค้าถึงจะแพ็คได้

Overview
เมนูการทำงานมีดังนี้
1. Order อ้างอิงจากโค้ดใน index.html ส่วนที่เกี่ยวข้องกับกระบบจัดการออร์เดอร์ทั้งหมด
2. Account ตรวจสอบยอดเกินสำหรับอนุมัติโอนคืน และ ดูจำนวนโคต้าคงเหลือของเครดิตการตรวจสลิป Easyslip
2. ใบงาน (Export) อ้างอิงจากโค้ดใน index.html 
3. เมนู QC อ้างอิงจากโค้ด QC_TR_4.9.html โดยใช้นำเอาฟีเจอร์จากในไฟล์มารวมเข้ากับโปรเจกต์นี้เข้าด้วยกัน
เพิ่ม logic เมื่อตรวจ QC แล้วให้แผนกจัดสินค้าเห็นสถานะว่ารายการสินค้านั้น QC แล้วหรือยัง 
4. Packing อ้างอิงจาก เมนู เลือกใบงานที่ต้องการจัดของโค้ดใน index.html
5. ระบบจัดการสินค้า (Products) อ้างอิงจาก index.html
6. จัดการลายการ์ตูน อ้างอิงจาก index.html
7. รายงานยอดขายตามสินค้า ： ขอรายงานยอดขายแบบมืออาชีพแสดงสิ่งที่ CEO ควรต้องดูอย่างละเอียด
8. Setting Users Role : มีเมนูสำหรับปรับให้ User เห็นเมนูไหนได้บ้าง
